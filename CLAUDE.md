## Commands

```bash
npm run dev          # Start Vite dev server
npm run build        # TypeScript check + production build
npm run lint         # ESLint validation
npm run preview      # Preview built dist locally
npm run generate-api # Regenerate API client from schema.yml
```

The `generate-api` command regenerates everything under `src/client/` from `schema.yml`. Run it after any backend API changes. Never manually edit files in `src/client/` ‚Äî they are fully auto-generated.

## Environment

Create a `.env` file in the root with:

```
VITE_API_URL=http://localhost:8000
```

The app defaults to `http://localhost:8000` if not set.

## Architecture

### Provider Hierarchy (`src/App.tsx`)

```
QueryClientProvider
  AuthProvider         ‚Üê user session, logout
    BranchProvider     ‚Üê active branch selection (multi-branch tenancy)
      Toaster          ‚Üê Sonner toast notifications
      RouterProvider   ‚Üê React Router
```

Authentication uses HttpOnly cookies. The client in `App.tsx` is configured with `credentials: 'include'` and two interceptors: one to attach Django CSRF tokens, one to auto-refresh on 401 using `v1RefreshCreate`.

### API Client (`src/client/`)

All files in this directory are generated by `openapi-ts` from `schema.yml`:

- `sdk.gen.ts` ‚Äî raw API functions (e.g., `v1ProductsList`)
- `types.gen.ts` ‚Äî TypeScript interfaces
- `zod.gen.ts` ‚Äî Zod validation schemas
- `@tanstack/react-query.gen.ts` ‚Äî pre-built React Query hooks

Prefer the generated React Query hooks over calling SDK functions directly. Only reach for raw SDK functions when writing custom mutations.

### State Management

- **Server state:** TanStack React Query (caching, refetching, mutations)
- **Global app state:** `AuthContext` and `BranchContext` via React Context
- **Local UI state:** `useState` for modals, tabs, filters, pagination
- No Redux or Zustand ‚Äî the above three layers are sufficient

Branch selection is persisted to `localStorage` via `BranchContext`.

### Routing (`src/router.tsx`)

- Public: `/login`, `/register`
- `ProtectedLayout` ‚Äî guards all authenticated routes; redirects to `/login` if unauthenticated, to `/create-company` if no company exists
- `DashboardLayout` ‚Äî sidebar + header wrapper for the main app area
- Key routes: `/dashboard`, `/inventory`, `/sales`, `/accounts`, `/contacts`, `/analytics`, `/settings`

### Component Layers

1. **`src/components/ui/`** ‚Äî base UI primitives (Button, Modal, Input, etc.)
2. **`src/components/[domain]/`** ‚Äî feature-specific forms and lists (e.g., `ProductForm`, `CustomerList`)
3. **`src/pages/[domain]/`** ‚Äî route-level pages that compose domain components

### Forms Pattern

All forms use React Hook Form + Zod. Error messages are in Spanish. Dynamic field arrays (e.g., product selling units, sale line items) use `useFieldArray` from React Hook Form. Forms open in modal dialogs for CRUD operations.

### Mobile Responsiveness

The app targets both desktop and mobile. The `useMediaQuery` hook drives conditional rendering ‚Äî several pages (notably `SalesPage`) render entirely different components for mobile vs. desktop. The bottom navigation bar (`BottomNavigationBar`) replaces the sidebar on mobile.

### Utilities

- `src/lib/utils.ts` ‚Äî `cn()` helper (clsx + twMerge) for conditional Tailwind classes
- `src/hooks/useExchangeRates.ts` ‚Äî USD/VES exchange rate support (multi-currency)
- Excel export via `xlsx` package, used in inventory and analytics pages
